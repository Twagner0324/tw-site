The Process

A good product starts with a strong foundation. The more the better. Here are some of the things that I'm familiar with which can lend a hand towards creating that new workflow.

Personas

Defining our user Personas can allow us to understand their goals, experiences and needs.  These fictional characters based on aggregate information can assist us in identifying pain points that we may want to avoid.  They humanize a role allowing us to be more empathic to their needs.   What would Mandy do?  To create a workflow solution for Mandy we need to know who she is

Journey Maps

So a guy walks into a hospital.... The user journey is another method that is instrumental in plotting our Persona's story.  A day in the life of one of our users can help us determine the bottlenecks.  Many times the friction is caused by the steady build up of multiple factors.  When did they engage with these factors? The journey can allow us to focus in on an engagement touchpoint.  The more detail we can document the more solutions we can find for improving the customer experience. 

Workflow Diagrams

When its time to work with development, interaction flow diagrams can trace the user actions with the program interface. The scope of a project can be understood if we can visualize the complexities and dependencies.  At this point we understand the user journey, the flow diagram is the beginning of outlining the solution for effective project planning and grooming. 

Wireframes

Let's get some feedback.  Simple wireframe mockups are a simple solution to determine whether you're headed in the right direction. Low fidelity wireframes with or without click interaction can help us begin collecting user feedback.  We can identify areas for improvement.  Wireframes continue to be helpful even after pixel accurate comps are created.  They are a great tool for targeting specific use cases 

High Resolution Comparables

The high res comps also commonly known as mock-ups or pixel comps is where the front end developer and the designer work together fine tuning the user experience. We are properly documenting new patterns and leveraging existing. Deviations from the design system need to be justifiable and intelligent.   If they haven't already been defined,  desktop and mobile interactions at the various breakpoints should be created.  Margins, paddings, borders, fonts and colors are refined in the CSS. If possible these should always be defined at a theme level.  This can ensure that any future refactors can be addressed at the top level reducing tech debt.  

User Testing

Although displayed last in this line up User Testing certainly never happens last. Although a fully developed finished product brings back valuable data, recurring usability tests are the most valuable. User testing begins the life cycle in the development process. It should be done during initial research and before every major milestone. Whether its a User Interface or a product, you're looking to answer the following questions. Was my user able to complete a pre-defined task? Did they accomplish it with much difficulty and was the process intuitive? These general questions amongst others, in addition to a carefully scripted survey can provide unbiased quantitative or qualitative data that is well worth the investment.
