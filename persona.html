Omnicell Personas

Project Overview

The compiliation of the Omnicell Persona Landscape Project originated in my first year at Omnicell. It would be an introduction to a formal set of Personas for the Omnicell software dept. I was asked to facilitate the collaboration of all Agile Project managers to compile a full set that could be used to educate our software developers.

Problem Statements

Database mainenance and organizational skills were critical. How would I efficiently compile and cross reference the aggregate data from multiple APOs and mitigate a general consensus of a core set of user types? How would we capture the core set of responsibilities of a user type if they conflicted between two products?

My Role:UI Designer and Graphic Designer

Tools I Used

  • Adobe InDesign (layout)
  • Adobe Photoshop (image editing)
  • Agile Practices (sprint planning and grooming)
  • Visual Studio (backlog maintenance)
  • Microsoft Excel (database management)
  • Powerpoint (team and departmental presentations)

The Solution

There were 5 core teams. I set up workshops with each of the APOs. The workshop goal was to complete the first persona to familiarize the product owner with my objectives. A questionnaire was created that allowed me to collect details for each user type. After determining the hospital roles, we then determined their hierarchy based on daily and weekly usage. We then worked together post workshop to complete the full set.

The first set was presented to the 2nd APO and so on. Any conflicting details were mitigated per role.

The Results

The first landscape set consisted of 23 personas overlapping as Primary, Secondary, Tertiary and Quarternary level users across 5 product lines. It continued to grow thereafter with new acquisitions. I was the sole owner and maintained the project for software Mountain View. Educational material was created and presented to the development teams to be displayed within each scrum team meeting room.
